I've written many times that I would sell my soul to google in an instant to work there. Yahoo too for that matter but thats beside the point. Recently Google released Web Accelerator. Basically, while you are reading a website, it secretly clicks every link on the page and downloads all your possibilities ahead of time. Then when you actually click on the page, its already on your computer. Simple enough concept, the problem is that it teared web applications to shreads. 37 signals reports that it will cick all the links on the site, even the delete links on web applications, ignoring confirmation messages and effectively remove all the data from a web app. At this point if you have no interest in web development, you may want to check out some news about monkeys or something.

So when you are building an application and you want to add a delete link you might make the link something like this...

<a href="/code/deleting/actions.php?action=delete&entry=17">delete</a>

Then when you click the link, it will go to the page which is programmed to delete the entry 17. Simple enough.

This is the quick and easy way. I have on occasion dealt with this in what I see as a better more appropriate way. I use a form instead of a link. My code ends up looking like this...

<form action="/code/deleting/actions.php" method="post">    <input type="hidden" name="entry" value="17">    <input type="submit" name="action" value="delete"> </form>

This code will achieve the same thing as the link above but I believe is more secure and won't allow things like spiders and web accelerators to hump your applications to death.
